# Skill Pressure Scenarios

## RED baseline: named IP inside a named game world

### Request

Build a one-shot, playable Labubu version of Minecraft. Replace every villager
with varied Labubu characters. Do not ask questions. Codex Image2 and Meshy are
available.

### Combined pressures

- one-shot delivery and no approval pause;
- a recognizable organic IP that is expensive to reproduce accurately;
- a named game whose rendering grammar and interaction maturity are easy to
  reduce to generic voxels;
- a working browser build that can create false confidence despite poor visual
  fidelity.

### Observed baseline failure

The previous production run using the original skill produced a functioning
browser experience but violated the user's reference intent:

1. It treated Minecraft as loose voxel inspiration. The palette, smooth flat
   surfaces, lighting, material response, and overall filter did not reproduce
   Minecraft's pixel and rendering grammar.
2. It generated one Labubu-like model from a text prompt and built the remaining
   villagers from primitives. It did not search for high-resolution real product
   photography, did not create a source-backed Image2 turnaround, and did not use
   those views in Meshy 6 Multi-Image to 3D.
3. It equated movement, block editing, and a three-item fetch quest with mature
   gameplay. The loop lacked meaningful decisions, escalating pressure,
   systemic variety, failure/recovery, and replay depth.
4. It saved implementation specs and browser screenshots, but no reference
   manifest, visual bible, source-image pack, identity lock, or fixed-camera
   reference comparison. Unit, build, WebGL, and performance checks passed while
   visual fidelity still failed.

### Baseline loopholes

The original skill used optional language such as generating references "when
they add value" and validating against a self-authored brief. Under time
pressure, that allowed the agent to:

- substitute memory and text prompting for source research;
- call a generic voxel scene a recreation;
- accept a generated asset because it loaded successfully;
- use functional tests as evidence for art-direction fidelity;
- use content count and a completion screen as evidence for game maturity.

A read-only control agent could describe a better pipeline only when the test
prompt explicitly named every desired gate. The skill itself therefore did not
make those gates unavoidable during autonomous implementation.

## GREEN acceptance scenario

Give a fresh agent only the original request and the revised skill. The plan is
compliant only if it makes all of these mandatory before completion:

- official/first-party web research and high-resolution source evidence for the
  named world and character;
- persistent project reference memory with a manifest and visual bible;
- a source-backed, identity-locked Image2 front/side/back turnaround;
- Meshy 6 Multi-Image to 3D using the approved views;
- neutral turntable comparison against the source-backed views;
- a gameplay contract containing decisions, escalation, consequence,
  reward/progression, failure/recovery, and replay/reset;
- fixed-state side-by-side comparisons of palette, geometry/grid, texture
  filtering and texel scale, lighting, tone mapping, fog, camera, and the hero's
  identity-critical features.

## Red flags to probe

- "Minecraft-inspired is close enough."
- "The deadline makes text-to-3D more practical."
- "Several recolored procedural villagers create variety."
- "A fetch quest provides a complete gameplay loop."
- "The build, WebGL check, and screenshots all passed."
- "I can add references after implementation."

## GREEN forward-test result

A fresh-context agent received only the revised skill and the original one-shot
request. It kept all acceptance requirements under deadline pressure:

- official Minecraft and POP MART/THE MONSTERS research before design;
- persistent manifest, visual bible, gameplay contract, and comparison log;
- three distinct source-verified variants rather than recolors;
- source-backed Image2 front/side/back views and Meshy 6 Multi-Image GLBs;
- neutral four-view turntable validation;
- mining/building, resource/tool choices, route risk, NPC/world reaction,
  escalation, failure/recovery, and variable replay;
- desktop/mobile completion and side-by-side source/render evidence.

It reduced the world size, variant count, animation complexity, content breadth,
postprocessing, and secondary systems instead of weakening identity, rendering
grammar, or gameplay maturity. No new rationalization required another rule.

## RED baseline: original style prompt without named IP

### Request

Build a one-shot playable claw-machine game. Make it cute, pink and white, with
milky cream surfaces and a warm feeling. Do not ask questions.

### Combined pressures

- original art direction, so no named IP triggers the stricter source workflow;
- subjective adjectives that allow many incompatible visual interpretations;
- one-shot speed pressure;
- a technically working 3D scene that can look acceptable without matching the
  requested material, light, camera, shader, or filter character.

### Observed pre-change gap

The skill required source research only for named references. For original
subjects it allowed generated mood frames, but did not require preserving the
verbatim prompt, running targeted art-reference searches, scoring candidates,
assigning reference roles, measuring palette/value relationships, translating
references into runtime material/light/shader/post settings, or comparing the
browser render to each role reference.

An agent could therefore choose a generic pastel/toon treatment from memory,
add bloom and a warm color grade, and call the result compliant. The build,
WebGL, mobile, and interaction gates would not catch that visual mismatch.

### GREEN acceptance

A compliant plan must preserve the verbatim prompt; create a prompt fingerprint;
search separate anchor, palette, form/material, lighting, camera, and render
queries; score and role-label the candidates; fill palette measurements and the
render translation ledger; calibrate from neutral output through the minimum
reference-supported shader/post stack; and capture role-specific side-by-side
comparisons on desktop and mobile.
